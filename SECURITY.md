# Security Policy

## Reporting Security Issues

If you find a security or privacy issue, please open a private report using the repository owner's preferred GitHub security reporting channel after the repository is published.

Do not post secrets, private drafts, unpublished manuscripts, review letters, datasets, or personally identifiable information in public issues.

## Data Handling

This repository contains documentation, Skill instructions, examples, and lightweight scripts. It does not require API keys or external services.

When contributing examples:

- remove author names, affiliations, and sensitive project identifiers;
- avoid unpublished manuscript content;
- use placeholders for datasets, metrics, and review metadata when needed.
